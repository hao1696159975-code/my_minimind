# 改进愿望清单（简历向 · 论文迁移）

> 原则：**先基线，后改进。**  
> 没有可复现 Dense mini 基线 + 测试，不接受「我改了 XX 论文结构」作为简历主叙事。

## 学习者动机（原话对齐）

- 风险：吃不透 / 只学皮毛  
- 目标：能把优质论文思想迁移到 MiniMind，并写进简历  

## 准入门槛（Capstone 开做前）

必须同时满足：

1. Phase 0–2 出门证（地图 + 数据契约意识 + 模型定位）  
2. Phase 4 Dense smoke：loss 合理、可 resume  
3. 至少一个自动化测试（shape 或 mask 或 ckpt）  
4. 能用 90 秒讲清「改动接口 / 如何验证 / 失败模式」

## 候选改进（按推荐优先级）

| ID | 改进 | 类型 | 依赖 Phase | 验证方式 | 简历一句话草稿 |
|---|---|---|---|---|---|
| I1 | Dataset loss-mask 可视化 + 单测 | 工程正确性 | 1,5 | 错 mask 对照 loss | 为教学型 LM 补齐监督信号回归测试 |
| I2 | 实验 run 管理（config→runs/→metrics） | 工程底座 | 0,4 | 同配置复跑曲线重合 | 自建小规模可复现训练实验台 |
| I3 | MoE expert load 日志 + collapse 探测 | MoE 可观测 | 7 | 极端路由构造 | 在原生 MoE 上实现路由可观测与负载诊断 |
| I4 | 可配置 LoRA target modules | 扩展点 | 6 | 参数量/merge 一致性 | 扩展简化 LoRA 注入面并做 merge 校验 |
| I5 | DPO logprob/mask 黄金测试 | 对齐正确性 | 8 | 手算小例子对齐 | 用最小用例锁定 DPO 实现符号 |

| I8 | 论文级：注意力/位置编码小变体（RoPE scale / YaRN 边界） | 研究迁移 | 2,3 | 长上下文失败模式 | 在小模型上验证位置外推边界 |
| I9 | OpenAI 兼容服务 + 模型卡 | 部署 | 11 | 压测/限流 | 完成训练到服务闭环 |

## 当前锁定策略

**主线 4–6 周标准路径**，Capstone 默认优先：

1. **I1 + I3**（正确性 + MoE 差异化，最好写）  
2. 若时间紧：I1 + I2  
3. 若论文欲望强且基线已过：I7（必须有对照表，禁止只改代码无数字）

## 禁止的简历写法

- 「完整复现 MiniMind」——区分度低  
- 「实现了 Transformer」——α 阶段已做过，不算本仓增量  
- 「loss 下降证明模型变强」——Day5 已否定  
- 「接入某论文」但无消融 / 无失败分析
